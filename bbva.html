Context

When financial apps become visually complex

Financial apps accumulate features over time. Interfaces become dense and visually competitive. Users often open banking apps with a single goal: quickly understand their finances.

This project explores how UX structure can simplify financial perception — without removing functionality.

How Might We

Reduce cognitive load in the banking interface so users can quickly understand their financial situation?

Key Insights

Three findings that shaped the redesign direction

01

Financial information lacked clear hierarchy

Balance, recent transactions and secondary UI components competed visually — making it harder to scan financial information quickly.

02

Transaction lists increased cognitive load

Dense layouts and minimal spacing made transaction scanning slower than necessary for routine financial checks.

03

Lack of interaction feedback reduced trust

Minimal visual confirmation when tapping elements created uncertainty during interaction — critical in financial contexts.

Product Opportunity

Restructure, don't add — improve the existing experience

Improve visual hierarchy

Make balance and key financial data immediately dominant in the visual structure.

Reduce cognitive load

Simplify transaction scanning through better spacing, grouping and typography hierarchy.

Strengthen interaction feedback

Add subtle micro-interactions that confirm user actions and build trust in every tap.

Product Principles

Three principles guiding every design decision

👁️

Clarity Over Density

Financial information must be immediately readable without visual competition.

🎯

Key Information First

Balance and recent transactions should dominate the visual hierarchy at all times.

✅

Feedback Builds Trust

Interfaces must confirm user actions clearly — especially in financial contexts.

Experience Focus

Two moments that matter most to users

The redesign focused on the two most frequent user intentions when opening a banking app.

1

Checking financial status

Balance and recent activity — the first thing users look for. The redesign makes this information visually dominant and scannable in under 3 seconds.

2

Exploring transaction details

Understanding specific transactions. The redesign reduces steps to access details and improves readability with better spacing and grouping.

Redesign Strategy

Three areas of structural improvement

📊

Clearer Visual Hierarchy

Balance and key financial data become visually dominant through size, weight and spatial positioning.

📋

Improved Transaction Readability

Better spacing, date grouping and typography hierarchy make scanning faster and more natural.

✨

Interaction Microfeedback

Subtle animations provide confirmation when interacting with transaction items and financial elements.

Microinteractions

Small details that build confidence

Tap transaction → Highlight on tap → Scale 1 → 0.98 → 1 → Smooth detail transition

In financial interfaces, microinteractions are not decorative — they are functional trust signals. Every tap that receives visual confirmation reduces uncertainty and increases perceived reliability of the product.
